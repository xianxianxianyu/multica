# Technology Stack

**Analysis Date:** 2026-04-13

## Languages

**Primary:**
- TypeScript ^5.9.3 — Frontend (web app, desktop app, shared packages). Strict mode enabled via `packages/tsconfig/base.json`.
- Go 1.26.1 — Backend server, CLI (`multica`), migration tool. Module path: `github.com/multica-ai/multica/server`.

**Secondary:**
- SQL — Database migrations and queries (raw SQL in `server/migrations/` and `server/pkg/db/queries/`, code-generated via sqlc).
- CSS (Tailwind v4) — Styling across all frontend apps and packages.

## Runtime

**Environment:**
- Node.js 22 — Required by CI (`.github/workflows/ci.yml`), Dockerfiles (`Dockerfile.web`), and development.
- Go 1.26.1 — Backend runtime, specified in `server/go.mod`.

**Package Manager:**
- pnpm 10.28.2 — Monorepo package manager. Specified via `packageManager` field in root `package.json`.
- Lockfile: `pnpm-lock.yaml` (present).
- Go modules — Backend dependency management, lockfile: `server/go.sum`.

## Frameworks

**Core:**
- Next.js ^16.1.6 — Web app framework (App Router). Config: `apps/web/`.
- Electron ^39.2.6 — Desktop app framework, built with electron-vite ^5.0.0. Config: `apps/desktop/`.
- Chi v5.2.5 — Go HTTP router. Config: `server/cmd/server/router.go`.
- React 19.2.3 — UI library, shared across all frontend packages. Pinned via pnpm catalog.

**State Management:**
- Zustand ^5.0.0 — Client-side state (Zustand stores in `packages/core/`).
- TanStack React Query ^5.96.2 — Server state (cache-first with WS-driven invalidation).

**Testing:**
- Vitest ^4.1.0 — Unit/integration tests for all TS packages.
- Playwright ^1.58.2 — E2E tests. Config: `playwright.config.ts`.
- Go standard `testing` — Backend tests.

**Build/Dev:**
- Turborepo ^2.5.4 — Monorepo task orchestration. Config: `turbo.json`.
- electron-vite ^5.0.0 — Electron dev/build pipeline. Config: `apps/desktop/`.
- electron-builder ^26.0.12 — Desktop app packaging.
- sqlc — Go code generation from SQL queries. Config: `server/sqlc.yaml`.
- GoReleaser v2 — Multi-platform CLI binary builds and Homebrew tap publishing. Config: `.goreleaser.yml`.

## Configuration

**Environment:**
- Root `.env` file (not committed) — loaded by `Makefile`. Contains DB URL, ports, secrets.
- Worktree `.env.worktree` — generated by `make worktree-env` for git worktree isolation.
- `docker-compose.selfhost.yml` defines all required env vars for self-hosting.

**Build Configuration Files:**
- `turbo.json` — Turborepo task pipeline (build, dev, typecheck, test, lint).
- `pnpm-workspace.yaml` — Workspace packages and catalog pinning.
- `packages/tsconfig/base.json` — Shared TypeScript config (strict mode, ESNext target, bundler module resolution).
- `packages/tsconfig/react-library.json` — Extends base with JSX and DOM types.
- `server/sqlc.yaml` — sqlc code generation config (PostgreSQL, pgx/v5 driver).
- `.goreleaser.yml` — GoReleaser binary build config (darwin/linux, amd64/arm64).
- `playwright.config.ts` — E2E test config (Chromium, 30s timeout).
- `packages/eslint-config/` — Shared ESLint config (base, react, next profiles).

**Key TypeScript compiler options** (from `packages/tsconfig/base.json`):
- `strict: true`, `noUnusedLocals: true`, `noUnusedParameters: true`
- `noImplicitReturns: true`, `noUncheckedIndexedAccess: true`
- `moduleResolution: "bundler"`, `target: "ESNext"`
